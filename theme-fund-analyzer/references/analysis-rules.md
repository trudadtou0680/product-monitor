# 分析规则

## 产品池

- 默认产品池为 `references/product-pools.md`。
- 每个主题使用二级标题 `## 主题名` 标识，下面用 Markdown 表格维护 `基金代码`、`基金名称` 两列。
- 每次用户提供新的产品池时，必须先调用 `scripts/fetch_theme_funds.py --theme <主题名> --set-pool <文件路径或-> --update-only` 更新 `references/product-pools.md`；同名主题直接覆盖，新主题直接新增。
- 用户产品清单可使用 Markdown 表格、CSV/TSV 风格文本或每行“基金代码 基金名称”/“基金名称 基金代码”；表格按 `基金代码`、`基金名称` 列解析。写入产品池的最终记录必须包含 6 位基金代码；名称-only 行必须先反查公开基金目录，能唯一匹配代码则写入，不能唯一匹配则跳过并提醒用户，不得让单行失败阻断其他有效产品写入。
- 用户提到的主题已存在于本地产品池时，必须优先使用本地产品池。
- 用户提到的是全新主题时，必须优先通过东方财富概念板块接口匹配细分概念板块，并记录概念板块名称、代码和成分股数量。
- 东方财富概念板块接口返回股票成分股，不直接返回完整基金产品池；匹配到概念板块后，优先使用东方财富板块主题基金接口返回的基金列表生成临时基金池。
- 若东方财富板块主题基金接口无返回，再使用概念板块名称和用户主题关键词在天天基金全量基金目录中生成临时基金池。
- 若无法唯一匹配东方财富概念板块，或匹配到概念板块但无法形成临时基金池，提示用户提供该主题基金产品池，不得用宽泛关键词硬凑基金名单。
- 产品池以基金代码为事实源；基金名称用于展示、辅助校验和代码缺失时的降级匹配。
- 基金代码能在公开基金目录中命中时，直接以代码对应产品作为分析对象；若代码对应名称与参考文件名称不一致，保留代码对应产品并在异常标记中列出 `名称代码不一致，已按代码优先`。
- 基金代码缺失、格式无效或无法匹配公开目录时，才使用基金名称匹配；若名称仍无法唯一匹配，该产品不进入收益、回撤和排名计算。
- 使用基金名称匹配时，若名称显式带有份额类别，如 A、C、E、I、Y、D，必须优先选择同份额候选；无法唯一匹配同份额候选时再留空。

## 时间区间

- `latest-day`：优先使用天天基金最新日涨幅；缺失时使用最近两个有效净值日计算。
- `1w`、`1m`、`3m`、`6m`、`1y`、`ytd`：优先使用天天基金阶段涨幅；缺失时分别按 7、30、91、182、365 个自然日和年初至今的累计净值序列兜底计算。
- `custom`：使用用户指定 `--start YYYY-MM-DD --end YYYY-MM-DD`。
- 自定义日期区间使用累计净值序列计算；若起止日不是交易日，使用不晚于目标日期的最近有效净值日。

## 多份额

- 同一产品不同份额以标准化基金名称聚合，去除末尾份额类别标识，如 A、C、E、I、Y、D。
- 同一产品不同份额在结果表中只能展示一行，不得因 A/C/E 等份额重复占用排名或 TOP N 名额。
- 同一产品存在 A 类份额时，展示代码、展示名称、收益率、最大回撤和名单内排名统一使用 A 类份额；若无 A 类份额，使用产品池中可匹配的首个有效份额。
- 合并规模对同一产品所有可匹配份额求和。

## 排序与排名

- 默认按区间收益率降序排序，收益率越大排名越靠前。
- 支持 `return`、`drawdown`、`scale` 三类排序字段。
- 最大回撤以负数展示；按回撤排序时默认数值越高越靠前，即回撤较小优先，越接近 0 越靠前。
- 排名只在同一主题池内有效。多主题输出时，每个主题独立排名。
- 无法计算排序字段的产品不参与排名，进入“待确认产品”或“数据缺口”。

## 表格输出

- 默认输出必须以脚本 stdout 中的 Markdown 表格为主体结果。用户没有明确指定输出结构时，不得把结果改写为 Top5、奖牌排名、项目符号列表、段落观察或纯摘要。
- 用户无特殊要求时，默认结果表列为 `排名`、`基金代码`、`基金名称`、`合并规模(亿元)`、`近1日(MM/DD)/排名`、`近1周/排名`、`近1月/排名`。
- 用户无特殊要求时，必须在每个主题的默认板块排名表之前置顶展示该主题中所有南方基金产品；置顶表与板块排名表使用相同列，`排名` 使用该主题全量产品实际排名，不得因置顶重新编号。
- 用户只提出“排名”“分析”“近1周/近1月前 N 只”“看表现”等泛化任务时，仍按默认表格输出；这些表达不构成自定义输出结构。
- 只有用户明确要求非表格结构时，才允许省略默认表格。
- 若已运行脚本，必须复制脚本 stdout 中的表格，不得重新手写榜单、抽取 Top5、按规模另列 bullet 或自行组织“观察”替代表格。
- 近1日列头必须标注实际最新净值日期，格式为 `MM/DD`。
- 近1日、近1周、近1月收益率必须带各自指标的名单内排名，展示为 `数值 / 排名`。
- 用户明确要求回撤时，回撤必须带回撤指标排名，展示为 `数值 / 排名`。
- 合并规模列头必须标注单位 `(亿元)`。
- 排名口径必须与当前指标一致：收益率排名按收益率降序，回撤排名按回撤较小优先。

## 输出与合规

- 输出必须标注数据来源和数据时点。
- 不得把历史收益表述为未来收益承诺。
- 不得使用“保本”“确定上涨”“必然优于”等确定性投资结论。
- 缺失数据必须明确列出，不能用均值、同类产品或人工估计填补。
